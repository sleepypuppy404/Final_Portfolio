Dog adoption planner





Context

One of the hardest things to plan for properly is adopting a dog. Dogs all require different amounts of time, of care, adopting of attention. However, owning a dog is also one of the greatest experiences possible, and can form a lifelong bond if done properly. However, doing so requires massive amounts of time, effort, and money, as well as investments to both train and feed the dog. In short, dogs take effort. As I am currently planning to adopt a puppy, and my family plans to adopt another dog, I thought of this problem, and realized that a lot of this research boils down to just a few numbers...

People Research

While researching these loveable pups, I also researched the factors people looked in when adopting a puppy. In general, to new dog owners, the most important aspects are as follows:
  1. How friendly is the dog, both with humans and other dogs
  2. How much does the dog cost
  3. How active is the dog.

Using this data, I created a dataset tracking this information, which would be used for the rest of the calculations done in this project.



Design

Dog Display System

The next part of the design was to actually consider the components to add. Since I needed to give the user the options to add dogs, I needed to create a method to store and display the dogs. To do so, I made a react component class that allowed me to display the dogs. This was a rather simple class, which was the goal with this asthetic. Simply a button to add the dog to the "puppy family" and see the basic information. These buttons were managed through react, each of which had a onClick handler that resulted in the incrementation of different values.

An example of three of the puppy react components side by side

Filtering/Sorting System

The next part of the design was to actually consider a method to narrow down the options available to the user. In order to make this easier, I added the option to sort (based on the cateogires of Friendliness, Activity, and Price due to research) and to filter (Based on Price and on Breed type, which while not as important during research, came up a lot). These components (shown below) were implemented using basic React functionality, changing state variables and then applying filters/sorting methodlogies based on what these state variables were.

The sorting/filtering interface

Family Display System

Finally, I needed a method to actually display what the puppy family would look like. Sticking with the very minimalistic asthetic, this family planner displays the basic cost, friendliness, and activity of the pets, while also maintaining a vary simple approach. All data here is actually derived from other react state variables, and is recalaculated every time a button is pressed. As for the add/subtract buttons, those have an onclick that decrements all values and recalculates.

The family planner (Shown without any dogs in it (Left) and with some in it (Right))




Wrapup

This project was extremely useful in teaching me how to utilize React, and how to make a minimalist design asthetic that would hopefully be easy to learn for new users. Takeaways

Lessons Learnt

  • React is an extremely useful tool
  • Syntax can be especially finicky in web languages, so be careful
  • Console logging is great for debugging

Insights Gained

  • Sometimes less is more when it comes to asthetic
  • Use of unconventional approaches like a sliding scale for price can add personality
  • It can be worthwhile to generalize components for cases when there could be many different ones
